# K1智能硬件全栈自动化测试系统
## 系统定位
以Mac为统一控制中心，通过USB连接K1安卓设备，实现跨端一体化自动化测试，无需人工操作即可完成全场景测试，自动生成标准化测试报告归档到Obsidian知识库，支持语义检索历史测试数据。
---
## 核心能力矩阵
### 🖥️ Mac控制端能力（无需操作设备）
1. 🔨 **远程构建编译**：自动从Git仓库拉取K1项目代码，调用远程编译服务/CI接口构建Android APK，编译完成后自动下载安装包，本地空间不足时自动清理旧编译缓存，不需要占用本地大量磁盘空间
2. 🔊 **音频输出**：自动播放测试音频/模拟人声指令（唤醒词、控制命令、标准测试音）
3. 🎙️ **音频采集**：调用Mac麦克风录制K1设备输出音频，自动检测音量/破音/杂音/响应延迟/语音识别准确率
4. 🎬 **视频输出**：自动播放测试视频/纯色画面/动效素材/校准图
5. 📷 **视觉采集**：调用Mac摄像头自动拍摄K1设备屏幕画面，校验显示效果/花屏/偏色/漏显示/动效流畅度/音画同步
6. ⚙️ **环境准备**：自动安装依赖（adb/ffmpeg/sox等）、检测USB连接状态、设备电量/温度/网络状态前置校验、自动清理测试垃圾文件
### 📱 K1设备端能力（ADB控制）
1. 📦 **APP管理**：安装/卸载/启动/停止/清空测试包
2. 📝 **日志收集**：实时抓取logcat、ANR/Crash/异常栈、系统日志
3. 🖱️ **UI自动化**：模拟点击/滑动/输入/按键、全页面遍历冒烟测试、Monkey随机压力测试
4. 📸 **界面采集**：设备端截图、录屏、UI层级导出
5. 📊 **性能采集**：CPU/内存/帧率/功耗/启动耗时/页面加载延迟数据采集
### 🧪 测试场景能力
1. 🎨 **UI还原度测试**：自动截图设备页面，自动拉取对应Figma设计稿做像素级对比，标注差异点，输出还原度得分报告
2. 🛡️ **功能稳定性测试**：长稳压力测试，自动统计ANR/Crash率、内存泄漏、卡顿率、资源占用，输出稳定性报告
3. 🎤 **语音交互测试**：自动播放唤醒词/语音指令→等待设备响应→采集音频/录屏/抓日志→自动校验唤醒成功率/识别准确率/响应延迟/回答正确性
4. 🎵 **音频播放测试**：自动播放标准测试音→采集设备输出→检测音量/失真/杂音/左右声道/延迟
5. 🎥 **视频显示测试**：自动播放测试视频→拍摄设备屏幕→检测花屏/偏色/卡顿/音画同步
6. 🔋 **功耗测试**：自动测试待机/播放/语音交互场景功耗数据
### 📂 结果归档
所有测试截图/音视频/日志/Markdown报告自动保存到Obsidian库路径：`02 执行项目/自有AI产品线/K1/测试结果/`，按日期分类，自动被Copilot/Smart Connections索引，支持语义检索历史bug、测试报告、问题记录。

## 使用方式
直接用自然语言描述测试需求即可自动执行，示例：
1. "安装~/Downloads/k1_v1.2.apk，跑首页UI还原度测试，对比Figma设计稿https://www.figma.com/xxx"
2. "做30分钟语音唤醒测试，播放100次唤醒词'你好鲁米'，统计唤醒成功率和响应延迟"
3. "跑2小时稳定性测试，遍历所有页面，统计崩溃率和内存泄漏"
4. "播放~/test_audio/test.mp3，检测K1扬声器输出有没有破音杂音"
5. "播放~/test_video/color_bar.mp4，检查屏幕显示是否正常有没有花屏偏色"
## 自动处理逻辑
- 自动检测依赖环境，缺失自动安装
- 自动检测设备连接状态，多设备时提示选择
- 测试结果自动按时间戳命名归档
- 崩溃/ANR自动高亮提取错误栈
- 异常情况自动截图/录屏留证
- 所有输出自动关联对应Obsidian测试笔记，可语义检索
